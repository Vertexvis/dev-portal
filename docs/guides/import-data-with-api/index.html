Import data

Vertex's Parts Library is a repository of your team's or organization's parts, each with references to geometry and optional metadata.

To import your 3D data into the Parts Library, do the following:

  • Create a file resource and upload your 3D data to it. Your files may contain either individual parts or larger assemblies.
  • Initiate a translation to create parts from the data contained in the file.

Uploading files#

See our supported file formats. If you do not have 3D data handy, you can download vertex-valve.jt to follow along. To create files in one of these formats, POST to the createFile API.

note
To import an entire assembly, you may need to take an additional step to prepare it for Vertex.
  • Request
  • Response
curl --location --request POST \
--url "https://platform.vertexvis.com/files" \
--header "Authorization: Bearer [YOUR_ACCESS_TOKEN]" \
--header "Content-Type: application/vnd.api+json" \
--data '{
"data": {
"attributes": {
"name": "vertex-valve.jt"
},
"type": "file"
}
}'
{
"data": {
"attributes": {
"created": "2021-01-01T12:00:00.000Z",
"name": "vertex-valve.jt",
"status": "pending"
},
"id": "[YOUR_FILE_ID]",
"type": "file"
}
}

Now that the file resource exists, POST to the uploadFile API.

  • Request
  • Response
curl --location --request POST \
--url "https://platform.vertexvis.com/files/[YOUR_FILE_ID]" \
--header "Authorization: Bearer [YOUR_ACCESS_TOKEN]" \
--header "Content-Type: application/octet-stream" \
--upload-file "[YOUR_PATH_TO_3D_DATA_FILE_INCLUDING_FILE_EXTENSION]"

204: No Content

Creating parts#

To make querying for parts easier, Vertex supports supplied IDs. You supply these IDs, likely using existing part and revision IDs from your PLM system. To initiate translations, POST to the createPart API. This API is asynchronous, returning the status of a queued translation.

  • Request
  • Response
curl --location --request POST \
--url "https://platform.vertexvis.com/parts" \
--header "Authorization: Bearer [YOUR_ACCESS_TOKEN]" \
--header "Content-Type: application/vnd.api+json" \
--data '{
"data": {
"attributes": {
"suppliedId": "[YOUR_SUPPLIED_ID]",
"suppliedRevisionId": "[YOUR_SUPPLIED_REVISION_ID]"
},
"relationships": {
"source": {
"data": {
"id": "[YOUR_FILE_ID]",
"type": "file"
}
}
},
"type": "part"
}
}'
{
"data": {
"attributes": {
"status": "running"
},
"id": "[YOUR_QUEUED_TRANSLATION_ID]",
"type": "queued-translation"
}
}

To check the status of queued translations, GET the getQueuedTranslation API. The response returns one of the following:

  • The status if running.
  • An error (similar to the response above).
  • Upon completion, redirects to the created part.
note

The PART_ID returned is for the root assembly or part. If your file contains a hierarchy, query for other parts using the getParts API.

  • Response

--header "Authorization: Bearer [YOUR_ACCESS_TOKEN]" \
--location
