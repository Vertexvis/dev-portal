Authentication

note
The following instructions assume:
  • You created a free developer account.
  • You created credentials in the Vertex Console and have your Vertex client ID and secret.

Gain secure access to the Vertex APIs by exchanging OAuth 2.0 credentials for an access token.

The flow is as follows:

  • Your application makes a secure HTTP call, providing its credentials. You receive an access token in response.
  • When making subsequent Vertex API calls, your application includes the access token.
sequenceDiagram participant CF as Client App Frontend participant C as Client App participant V as Vertex API clients participant API as Vertex API CF->>+C: Authenticate C->>+API: Request token with credentials API-->>-C: Token response C-->>-CF: Token CF->>+V: API client call with token V-->>-CF: API client response C->>+API: API request with token API-->>-C: API response

Creating access tokens​

To create an access token, POST to the createToken API with your credentials. The token API uses HTTP basic access authentication. The request contains the Authorization: Basic [YOUR_CREDENTIALS] header, where YOUR_CREDENTIALS is the Base64 encoding of your Vertex client ID and secret joined by a colon. The following command performs this encoding and creates an access token.

note
Your Vertex client secret is sensitive and must be kept secure at all times. You should never include it in client-side web applications.
  • Request
  • Response
curl --location --request POST \
  --url "https://platform.vertexvis.com/oauth2/token" \
  --header "Authorization: Basic $(echo -n "[YOUR_CLIENT_ID]:[YOUR_CLIENT_SECRET]" | base64)" \
  --header "Content-Type: application/x-www-form-urlencoded" \
  --data-urlencode "grant_type=client_credentials"
{
  "access_token": "[YOUR_ACCESS_TOKEN]",
  "account_id": "[YOUR_ACCOUNT_ID]",
  "expires_in": 3600,
  "scopes": [],
  "token_type": "bearer"
}

Using access tokens​

Access tokens are valid for one hour. Include them as a bearer token in the Authorization header for subsequent API calls.

curl --location --request GET \
  --url "https://platform.vertexvis.com/files" \
  --header "Authorization: Bearer [YOUR_ACCESS_TOKEN]"

Viewer component​

The Viewer component authenticates using stream keys. These keys grant access to specific scenes, so you'll need one first. If you don't have one, create one by following the Import data and Render static scenes guides. Then, to create a stream key, POST to the createSceneStreamKey API.

  • Request
  • Response
curl --location --request POST \
  --url "https://platform.vertexvis.com/scenes/[YOUR_SCENE_ID]/stream-keys" \
  --header "Authorization: Bearer [YOUR_ACCESS_TOKEN]" \
  --header "Content-Type: application/vnd.api+json" \
  --data '{
  "data": {
    "attributes": {
      "expiry": 600
    },
    "type": "stream-key"
  }
}'
{
  "data": {
    "attributes": {
      "key": "[YOUR_STREAM_KEY]",
      "expiry": 600
    },
    "id": "[YOUR_STREAM_KEY_ID]",
    "relationships": {
      "scene": {
        "data": {
          "id": "[YOUR_SCENE_ID]",
          "type": "scene"
        }
      }
    },
    "type": "stream-key"
  }
}

Now, pass the stream key to the viewer.load method in the Viewer component.

async function main() {
  const viewer = document.querySelector('vertex-viewer');
  await viewer.load('urn:vertex:stream-key:[YOUR_STREAM_KEY]');
}
