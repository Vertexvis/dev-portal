Import metadata

You can optionally import your team's or organization's metadata properties while importing data and add additional properties to existing part revisions.

Indexing during part creation​

While creating parts, set the indexMetadata request body parameter. This instructs Vertex to extract and index metadata properties from your 3D data.

{
  "data": {
    "attributes": {
      "indexMetadata": true
      // ...other attributes...
    },
    "relationships": {
      // ...relationships...
    },
    "type": "part"
  }
}

Adding to existing part revisions​

To add metadata to existing part revisions, PATCH the updatePartRevision API.

  • Request
  • Response
curl --location --request PATCH \
  --url "https://platform.vertexvis.com/part-revisions/[YOUR_PART_REVISION_ID]" \
  --header "Authorization: Bearer [YOUR_ACCESS_TOKEN]" \
  --header "Content-Type: application/vnd.api+json" \
  --data '{
  "data": {
    "attributes": {
      "metadata": {
        "MY_STRING_PROPERTY": {
          "value": "PN12345",
          "type": "string"
        },
        "MY_LONG_PROPERTY": {
          "value": "12345",
          "type": "long"
        },
        "MY_FLOAT_PROPERTY": {
          "value": "123.45",
          "type": "float"
        },
        "MY_DATE_PROPERTY": {
          "value": "2021-01-01T12:00:00.000Z",
          "type": "date"
        }
      }
    },
    "id": "[YOUR_PART_REVISION_ID]",
    "type": "part-revision"
  }
}'
{
  "data": {
    "type": "part-revision",
    "id": "[YOUR_PART_REVISION_ID]",
    "attributes": {
      "created": "2021-01-01T12:00:00.000Z",
      "suppliedId": "[YOUR_SUPPLIED_REVISION_ID]"
    },
    "relationships": {
      "part": {
        "type": "part",
        "id": "[YOUR_PART_ID]"
      }
    }
  }
}

Use the ISO 8601 representation for dates. Most languages provide a native mechanism for it.

  • JavaScript/TypeScript
  • C#
  • Java/Scala/Kotlin
new Date().toISOString();
// 2021-01-01T12:00:00.000Z
DateTime.UtcNow.ToString("o", System.Globalization.CultureInfo.InvariantCulture)
  // 2021-01-01T12:00:00.0000000Z
java.time.Instant.now().toString();
  // 2021-01-01T12:00:00.000Z

Getting metadata​

To get metadata for a part revisions, GET the getPartRevision API. By default, the response does not include metadata. You must explicitly request it using the fields[part-revision]=metadata query parameter.

note

This API is eventually consistent. If you just added or updated metadata, it may take time for the response to reflect changes.

  • Request
  • Response
curl --location --globoff --request GET \
  --url "https://platform.vertexvis.com/part-revisions/[YOUR_PART_REVISION_ID]?fields[part-revision]=metadata" \
  --header "Authorization: Bearer [YOUR_ACCESS_TOKEN]" \
{
  "data": {
    "type": "part-revision",
    "id": "[YOUR_PART_REVISION_ID]",
    "attributes": {
      "metadata": {
        "MY_FLOAT_PROPERTY": {
          "value": "123.45",
          "type": "float"
        },
        "MY_LONG_PROPERTY": {
          "value": "12345",
          "type": "long"
        },
        "MY_STRING_PROPERTY": {
          "value": "PN12345",
          "type": "string"
        },
        "VERTEX_PART_NAME": {
          "value": "STOP VALVE ASSEMBLY",
          "type": "string"
        },
        "MY_DATE_PROPERTY": {
          "value": "2021-01-01T12:00:00.000Z",
          "type": "date"
        }
      }
    },
    "relationships": {
      "part": {
        "type": "part",
        "id": "[YOUR_PART_ID]"
      }
    }
  }
}
