Postman quick start

For advanced functionality beyond what our UI Components provide, you can optionally integrate directly with our API. The API guides show examples using curl. If you're less comfortable with the command line, though, Postman is a better choice. If you'd rather jump staight to calling our API in your application, check out our API Clients instead.

Postman is a tool to build and test HTTP requests in an easy-to-use interface without configuring a full development environment. Vertex provides a Postman Collection, a set of pre-configured API requests getting you started with the API without manually configuring each request.

Open Vertex's Postman collection​

  1. Download and install Postman.
  2. In the upper-right corner of our API Reference documentation, click Run in Postman.

The collection contains the reference documentation as well, as shown below.

Notes about the Postman collection and API Reference code samples,

  • They include all required and optional body parameters for completeness. Remove any optional parameters as desired.
  • They use auto-generated IDs and other values that may share the same value for ease of documentation only. In actual requests and responses, the IDs should uniquely identify their corresponding resource.

Get an access token​

You gain secure access to Vertex APIs by exchanging OAuth 2.0 credentials for an access token. Subsequent Postman calls will then include the access token in the Authorization header automatically.

note
The following instructions assume:
  • You created a free developer account.
  • You created credentials in the Vertex Console and have your Vertex client ID and secret.
  1. In Postman, on the left pane, right-click Vertex Platform API collection.
  2. Click Edit.

  1. In the Edit Collection dialog, click the Authorization tab.
  2. On the left, set Type to OAuth 2.0.
  3. On the right, from Grant Type, select Client Credentials.
  4. Enter the Vertex Client ID and Client Secret that you received when you created your Vertex account.

  1. Click Get New Access Token.
  2. Click Proceed and then Use Token on the next two dialogs.
  3. In the bottom right corner, click Update.
note
Your Vertex client secret is sensitive and must be kept secure at all times. You should never include it in client-side web applications.

Congratulations! You got an access token and are ready to make API calls.

note

Access tokens expire in 60 minutes to help keep your data safe. If you receive a 401: Unauthorized error, it may indicate an expired access token. Perform the steps above to get a new one.

Make API requests​

Select a request from the Vertex Platform API collection. In this example, we use get Files in the files folder.

The Params tab (and Body tab if this were a POST or PATCH request) include all required and optional parameters for completeness. They may also use auto-generated values, as shown below.

  1. Unselect the checkbox next to each parameter to use the defaults.
  2. On the Authorization tab, for Type, select Inherit auth from parent.
  3. Click Send to the right of the request URL, and you'll see the response in the bottom pane.

The data array in the response is empty since we haven't created any files yet. Let's change that by following the Import data guide.
